# PRD: kyudo-dojo-hub

**作成者**: Ryusei | **日付**: 2026-03-30 | **ステータス**: ドラフト
**アイデアID**: 493 | **ドメイン**: 弓道 | **難易度**: Medium

## 問題定義

弓道の射形評価は師範の主観に依存しており、稽古者が自分の射形を客観的に把握する手段が限られている。また、道場運営（的場予約・会費管理・会員管理）と個人の稽古記録・段位審査準備が別々のツールやアナログ管理に分散しており、一気通貫のデジタルプラットフォームが存在しない。射法八節の「型」を定量化し、精神修養と技術向上の両面を可視化することで、スコアだけを追うアーチェリーアプリとは決定的に差別化する。

## 目標と成功指標

| 目標 | 指標 | ターゲット |
|------|------|----------|
| 射形の定量分析を実現する | 骨格推定による八節フェーズ自動分割の精度 | 正解率80%以上（10件のサンプルデータ） |
| 稽古記録の継続率を高める | 週1回以上記録するアクティブユーザー率 | 60%以上（MVP期間中） |
| 道場運営の効率化 | 的場予約・会費管理のデジタル化完了率 | 1道場でフル運用 |
| 段位審査の準備を支援する | 審査チェックリスト完了率 | 利用者の70%がチェックリストを活用 |

## スコープ外

- リアルタイム動画ストリーミング解析（MVPではアップロード後のバッチ処理のみ）
- 複数道場間のランキング・対戦機能
- モバイルネイティブアプリ（MVP はレスポンシブ Web）
- 課金・決済機能
- ML モデルの独自学習（MVP は MediaPipe + ルールベース分類）

## ユーザーストーリー

- **弓道稽古者**として、稽古動画をアップロードして射法八節の各段階のスコアリングを見たい。なぜなら自分の射形の弱点を客観的に把握して改善したい。
- **弓道稽古者**として、稽古日誌に的中率・気づき・師範コメントを記録したい。なぜなら成長の軌跡を振り返り段位審査に備えたい。
- **道場管理者**として、的場の予約カレンダーと会員管理ダッシュボードを使いたい。なぜなら紙や口頭での管理を効率化したい。
- **段位審査受験者**として、審査項目チェックリストで準備状況を管理したい。なぜなら審査までに何を改善すべきか明確にしたい。

## 技術要件

### 技術スタック

- **フロントエンド**: TypeScript / React（Vite）
- **API層**: Go（gRPC / REST gateway）
- **ML/データ処理**: Python（MediaPipe 骨格推定）
- **データベース**: PostgreSQL
- **キャッシュ**: Redis
- **インフラ**: GCP Cloud Run + Cloud Storage（動画保存）

### アーキテクチャ

React SPA -> Go API (gRPC) -> PostgreSQL/Redis
                  |
                  v
          Python 処理エンジン (MediaPipe)
                  |
                  v
          GCP Cloud Storage (動画)

- フロントエンド: React SPA（Vite ビルド）
- API: Go gRPC サーバー + REST gateway（grpc-gateway）
- ML: Python MediaPipe ワーカー（非同期ジョブ）
- 通信: フロントエンド <-> API は REST、API <-> ML は gRPC

### 非機能要件

- パフォーマンス: 動画アップロード後30秒以内に骨格推定結果を返却
- セキュリティ: 動画データは署名付きURLで保護、認証はJWT
- スケーラビリティ: 同時接続100ユーザー、動画処理キュー10件並列

## 競合分析

| 競合 | URL | ギャップ |
|------|-----|---------|
| Hudl / Coach's Eye | https://www.hudl.com | 汎用スポーツ動画分析。弓道の射法八節に特化した分析なし |
| Dartfish | https://www.dartfish.com | プロスポーツ向け高価格。道場運営機能なし |
| 弓道手帳（アナログ） | - | 紙ベース。動画分析・デジタル管理不可 |

**差別化ポイント**: 弓道特化の射形分析+道場運営+段位審査一気通貫は汎用スポーツアプリにない垂直統合

## 技術リスク

| リスク | 発生可能性 | 影響 | 軽減策 |
|--------|----------|------|--------|
| リアルタイム骨格推定の精度確保 | 高 | 高 | MVPではルールベース分類で開始、段階的にML精度を向上 |
| 動画処理の遅延 | 中 | 中 | 非同期キュー処理、進捗表示UIで体験を補完 |
| 弓道ドメイン知識の不足 | 中 | 高 | 射法八節の文献調査、弓道経験者へのヒアリング |

## マイルストーン

### Phase 0: MVP（2週間）

1. ルールベースの分類/推定（ML前段階）
2. データ入力フォーム + 結果表示UI
3. 10件のサンプルデータでの動作検証

## マネタイズ

弓道特化のデジタルツールは皆無。道場単位のサブスクリプション（月額制）と個人向けフリーミアムモデルの組み合わせが有望。
